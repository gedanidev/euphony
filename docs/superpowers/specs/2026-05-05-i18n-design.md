# i18n Multi-idioma — Euphony

## Contexto

La app está completamente en español. Se quiere agregar soporte para Español, Inglés y Japonés usando `react-i18next`, con selector de idioma en la página de Ajustes y persistencia en `localStorage`.

---

## Alcance (fase 1)

Solo partes principales de la UI:
- Navegación lateral (Layout.jsx)
- Títulos de páginas y botones principales
- Tabs de disponibilidad (Todos / Disponibles / Wishlist / No disponibles)
- Mensajes de estado vacío y errores
- El selector de idioma en Settings.jsx

---

## Arquitectura

### Dependencias nuevas

```
react-i18next
i18next
```

### Estructura de archivos

```
frontend/src/i18n/
  index.js              ← inicialización de i18next
  locales/
    es.json             ← español (idioma por defecto)
    en.json             ← inglés
    ja.json             ← japonés
```

### Configuración (`i18n/index.js`)

- `lng` por defecto: `"es"`
- `fallbackLng`: `"es"` (si falta una clave en otro idioma, usa español)
- Detector: `localStorage` con key `euphony-language`
- Recursos: importados directamente (sin lazy loading — los archivos son pequeños)

### Integración en React

- Importar `i18n/index.js` en `main.jsx` (efecto de side-effect, antes del render)
- Usar `useTranslation()` hook en cada componente para acceder a `t('clave')`

---

## Selector de idioma

**Ubicación:** Settings.jsx, primera sección

**UI:**
```
Idioma / Language / 言語
[ Español ▼ ]
```

**Opciones:**
| Valor | Label |
|-------|-------|
| `es`  | Español |
| `en`  | English |
| `ja`  | 日本語 |

**Comportamiento:**
- Cambio instantáneo (sin recarga)
- Persiste en `localStorage` con key `euphony-language`
- Idioma por defecto: `es`
